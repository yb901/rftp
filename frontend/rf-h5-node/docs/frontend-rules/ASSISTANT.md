# H5 前端助手入口

本文是编码助手处理 `rf-h5-node` 公众号 H5 前端任务时的入口。先读取仓库根目录 `AGENTS.md`，再读取本文。

## 读取原则

- 优先沿用当前 `src/features`、`src/shared`、`src/views` 的组织方式。
- H5 使用 `React 18 + Ant Design Mobile 5 + Vite + TypeScript`。
- 环境变量使用 `VITE_*`，不要硬编码接口地址。
- 不允许散落业务硬编码值；状态、类型、渠道、开关、固定文案映射等值应抽成枚举、常量或映射函数。

## 业务流程

- 员工端以手机号登录为主，登录流程包含图形验证码和短信验证码。
- 绩效详情页必须清晰展示绩效描述、周期、确认截止时间、员工绩效和当前状态。
- 确认绩效需要二次确认，成功后给出明确结果反馈。
- 反馈入口只在确认截止时间前开放；超期后只展示状态和历史信息。
- 历史绩效允许查询，当前周期和历史周期应有明确区分。

## 安全与留痕

- 登录和确认相关接口调用要保留必要上下文，例如验证码 token、手机号、客户端环境信息。
- 前端不要展示或持久化短信验证码明文之外的敏感凭证。
- H5 本地存储只保存必要会话信息，到期或退出后清理。
- 所有关键写操作的失败原因要明确展示，避免员工重复误操作。

## UI 与适配

- 优先使用 Ant Design Mobile 组件。
- 适配公众号内置浏览器和常见移动端宽度。
- 关键按钮固定、醒目但不遮挡内容；长绩效描述和反馈内容必须可读。
- 避免营销式落地页，第一屏直接承载员工查看和处理绩效的主流程。

## 完成前检查

涉及前端代码改动时执行：

```bash
npm run build
```

如果依赖未安装，优先使用 `npm install` 或 `npm ci` 安装后再构建。
