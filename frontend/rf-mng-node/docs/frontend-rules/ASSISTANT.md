# 管理后台前端助手入口

本文是编码助手处理 `rf-mng-node` 管理后台前端任务时的入口。先读取仓库根目录 `AGENTS.md`，再读取本文。

## 读取原则

- 优先沿用当前页面、API 客户端和样式组织方式。
- 管理后台使用 `React 18 + Ant Design 5 + Vite + TypeScript`。
- 业务类型、枚举、接口请求优先集中到 `src/api.ts`；当模块复杂度上升后再拆分到 `src/features/{context}`。
- 用户当前明确要求优先于本文规则。

## 页面与交互

- 管理后台是运营工具，界面应清晰、紧凑、便于扫描和重复操作。
- 列表页必须包含明确筛选、分页、加载态、空态和错误提示。
- 危险操作、后台调整、批量导入等写操作需要二次确认或明确反馈。
- 后台调整、绩效导入、导出等流程要保留操作结果提示和错误明细展示入口。
- 不做营销式首页；第一屏应直接进入可操作业务界面。

## API 与类型

- 请求参数和响应类型使用 TypeScript 显式定义。
- 后端枚举值在前端集中维护 label、颜色和禁用逻辑，不散落硬编码。
- 日期时间展示和提交格式保持一致，避免在页面中到处手写格式化。
- 不硬编码生产接口地址，接口前缀通过 Vite 配置或运行环境注入。

## 样式

- 优先使用 Ant Design 组件和项目已有 CSS 变量。
- 表格、筛选区、弹窗和按钮尺寸保持管理后台风格，不使用过大的展示型排版。
- 避免卡片套卡片；页面区域使用清晰间距和分组。
- 文案必须能在常见桌面宽度下完整展示，不让按钮、表头、状态标签文字溢出。

## 完成前检查

涉及前端代码改动时执行：

```bash
npm run build
```

如果依赖未安装，优先使用 `npm install` 或 `npm ci` 安装后再构建。
